初试 Atom

为什么要用Editor

  • IDE语言针对性强不够灵活
  • 本人接触Web项目比较多，改文件，打开浏览器就可以，没有漫长编译的过程
  • 安装方便，打蚊子何须大炮

为什么是Atom

  • Vim 用过，Emacs在学
  • Atom 开源，可完全定制
  • Facebook，M$S 都基于atom开发了编辑器，大公司的选择不会太偏
  • 各种深度勾结github ，同一体验

奇技淫巧

Atom 是可以该键盘绑定的，进来都是用netbeans多，但是！我要用原生的，万一哥哪天迁移到了iOS。。。

  • 代理设置
    上网是根基啊，尤其是对这个Atom来说，很多package可装。并且鄙人很多时候是连的公司的网线，怎么能少了代理定制（当然可以设置IE代理曲线解决），
    但是Atom有啊，可以在用户目录里的 .apmrc 文件里修改

  • Marktown支持，markdown-preview 插件一装，快捷键一开 - C-S M，实时preview，其他的markdown tool可以卸载了。

  • Keyboard Resolver 功能，ctrl+’.’,底部会打开了监听按键面板，按了什么键对应什么功能都有,还带有历史记录功能。

写在最后

Atom 才刚刚发布1.0，开启速度是痛点，但是得到饿了社区的回馈，相信性能会越来越好，至少得和VSC差不多吧 ：）

Browsersunc初探

偶然间在搜索web测试的时候看到这么一篇牛气的的博客移动端前端开发调试-于江水,好奇Browsesync 这个神奇的工具，就开始了折腾之旅。

安装

推荐按照 browsersync中文网的教程安装，吐槽下，本人强迫症发作，全局安装方式遇到了各种坑，推荐同学们使用本地安装方式，就像教程里推荐的一样。

打开静态网页，默认打开所在目录index.html

打开动态网页

输入命令后，会回显一些Url信息，默认会在localhost:3000 代理原站点，会在localhost:3001生成web console界面如果部署的机器有2个ip，这个工具会默认选择第一个，官网也写有如何配置用于调试的外部url option。
