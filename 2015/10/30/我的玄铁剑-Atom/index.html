我的玄铁剑-Atom

  • atom
  • tool

为什么要用Editor

  • IDE语言针对性强不够灵活
  • Web项目比较多，该文件，打开浏览器就可以，没有漫长编译的过程
  • 安装方便，打蚊子何须大炮

why not Sublime or Brackets

  • Sublime太高冷，用的话还是开源的吧
  • Brackets 针对Web文件，哥怎么可能局限于此
  • Atom === github === open source 可装bility的空间大
  • Atom 是一支潜力股，做原始股东蛮好的
  • 各种深度勾结 git ，轻松管理版本和差异

奇技淫巧

Atom 是可以该键盘绑定的，进来都是用netbeans多，但是！我要用原生的，万一哥哪天迁移到了iOS。。。

  • 代理设置
    上网是根基啊，尤其是对这个Atom来说，很多package可装。并且鄙人很多时候是连的公司的网线，怎么能少了代理定制（当然可以设置IE代理曲线解决），
    但是Atom有啊，可以在用户目录里的 .apmrc 文件里修改

    proxy=http:// IP addr|| domain name :port
    ```

  • Marktown支持，markdown-preview 插件一装，快捷键一开，实时preview，其他的markdown tool可以卸载了。

  • Keyboard Resolver 功能，这个牛逼，ctrl+’.’ 一按,底部会打开了一个监听键盘面板，按了什么键对应什么功能都有,还带有历史记录功能。。
